Built for practical business communication

The goal of this prototype is simple: help local businesses handle more inbound calls without relying on missed voicemails, manual follow-up, or inconsistent response time.

01

Inbound call handling

The system is designed to answer incoming voice calls and return TwiML instructions for guided call handling.

02

SMS follow-up

Text message workflows can confirm receipt, provide next steps, or notify the business owner that a new lead has arrived.

03

Lead capture

Caller details and message events are structured for storage so follow-up can be tracked instead of lost.

How the workflow works

The current build focuses on a straightforward webhook-based flow that can later be expanded into a full service offering.

1

Incoming contact

A caller or texter reaches the Twilio number assigned to the workflow.

2

Webhook handling

The request is sent to a Flask backend endpoint such as /voice or /sms for processing.

3

Business follow-up

The system responds, stores the interaction, and prepares the next action for the business owner or customer.

Current project FAQ

A plain-language snapshot of what this project is and how it is being tested right now.

What is this website for?

This page documents a working prototype for an AI receptionist workflow that uses Twilio voice and SMS with a Python Flask backend.

Is this live in production yet?

No. The prototype is currently being developed and tested locally. It is not yet a public production service.

What types of messages are planned?

Example outbound SMS: Hello, this is the automated receptionist. We received your call and will contact you shortly.

Who is this intended for?

The prototype is being built for small service businesses that need a better way to capture calls, messages, and customer follow-up.
